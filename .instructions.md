# Framework Instructions & Architecture Guide
# JSON Parser 2.0 - Generic Denormalization Framework

## Problem Statement

When raw data (JSON/XML) is stored in a data warehouse like Trino/Iceberg, it often comes as:
- Unstructured nested payloads in single columns
- Multiple source formats (JSON, XML)
- Varying schema across payloads (schema drift)
- Repeated nested structures (1-to-many relationships)

**Goal**: Transform raw nested payloads into normalized relational tables suitable for analytics while:
- Handling schema drift automatically
- Extracting nested entities into separate tables
- Supporting upsert/merge operations
- Being reusable across many different tables
- Not requiring hardcoded logic per table

## Design Principles

### 1. Configuration-Driven
All table-specific logic lives in YAML configs, not Python code.
- `table_config.yaml`: Business key mappings
- `schema_definitions.yaml`: Optional schema presets
- No hardcoded table names or logic

### 2. Modular Architecture
Each concern has its own module:
- **config_loader.py**: Configuration management
- **trino_client.py**: Database connectivity
- **json_parser.py**: Format-specific parsing
- **schema_builder.py**: Type inference & drift handling
- **sql_builder.py**: DDL/DML generation
- **denormalizer.py**: Main orchestration

### 3. Automatic Schema Inference
- Type detection from first N rows
- No manual schema definition required (config optional)
- Schema drift: Add columns, never drop
- Nullable column tracking

### 4. Nested Structure Extraction
- Flat parent object → Parent table
- Nested arrays → Child tables (1-to-many)
- Nested objects → Linked via foreign key
- Automatic naming: `parent_table_child_entity`

### 5. Merge/Upsert Support
- Business key configuration per table
- Merge operation for updates
- No duplicate rows
- Idempotent execution

## Architecture Overview

```
┌─────────────────────────────────────────────┐
│  Raw Data Source (Trino/Iceberg)           │
│  - live25_raw_dev.events                   │
│  - Array of JSON payloads in "payload" col│
└────────────────┬────────────────────────────┘
                 │ Read raw rows
                 ▼
┌─────────────────────────────────────────────┐
│  Trino Client                              │
│  - Connection management                   │
│  - Query execution                         │
└────────────────┬────────────────────────────┘
                 │ Fetch payloads
                 ▼
┌─────────────────────────────────────────────┐
│  JSON Parser                               │
│  - Parse JSON strings                      │
│  - Flatten nested structures               │
│  - Identify nested entities                │
└────────────────┬────────────────────────────┘
                 │ Structured data
                 ▼
┌─────────────────────────────────────────────┐
│  Schema Builder                            │
│  - Infer types from values                 │
│  - Track nullable columns                  │
│  - Handle schema drift                     │
└────────────────┬────────────────────────────┘
                 │ Schema + Data
                 ▼
┌─────────────────────────────────────────────┐
│  SQL Builder                               │
│  - Generate CREATE TABLE DDL               │
│  - Generate INSERT/MERGE statements        │
│  - ALTER TABLE for drift                   │
└────────────────┬────────────────────────────┘
                 │ SQL statements
                 ▼
┌─────────────────────────────────────────────┐
│  Denormalizer (Orchestrator)              │
│  - Coordinate all components              │
│  - Load parent table                       │
│  - Load child tables                       │
│  - Handle errors & logging                 │
└────────────────┬────────────────────────────┘
                 │ Execute SQL
                 ▼
┌─────────────────────────────────────────────┐
│  Target Schema (Trino/Iceberg)            │
│  - live25_raw_denormalise.events (parent) │
│  - live25_raw_denormalise.events_*.* (children)
└─────────────────────────────────────────────┘
```

## Usage: Single Table

```bash
python main.py \
  --trino-host localhost \
  --trino-port 8080 \
  --trino-user admin \
  --trino-password password \
  --source-schema live25_raw_dev \
  --source-table events \
  --target-schema live25_raw_denormalise \
  --target-table events
```

## Examples: Nested JSON Transformation

### Example 1: Event with Attributes
**Input** (live25_raw_dev.events):
```json
{
  "id": 123,
  "event_type": "booking",
  "timestamp": "2024-03-13",
  "attributes": [
    {"key": "source", "value": "web"},
    {"key": "channel", "value": "mobile"}
  ]
}
```

**Output Tables**:
- `live25_raw_denormalise.events`: (id, event_type, timestamp)
- `live25_raw_denormalise.events_attributes`: (parent_id, key, value)

### Example 2: Order with Line Items and Addresses
**Input** (live25_raw_dev.orders):
```json
{
  "id": 456,
  "customer_id": 789,
  "status": "confirmed",
  "line_items": [
    {"product_id": 1, "qty": 2, "price": 99.99},
    {"product_id": 2, "qty": 1, "price": 49.99}
  ],
  "shipping_address": {
    "street": "123 Main St",
    "city": "Seattle"
  }
}
```

**Output Tables**:
- `live25_raw_denormalise.orders`: (id, customer_id, status, shipping_address_street, shipping_address_city)
- `live25_raw_denormalise.orders_line_items`: (parent_id, product_id, qty, price)

## Known Limitations (Phase 1)

- Single-level nesting only (will extend to N-levels)
- No XML support yet
- No incremental sync
- No data quality validation
- Simple type inference (no domain types)
